## 根目录配置文件详解

### 1. `!setup.txt` — 主服务器配置

- **格式**: INI 风格（`[Section]` + `Key=Value`）
- **大小**: 81,181 字节，约 3,565 行
- **编码**: GBK/GB2312

**主要 Section**：
```ini
[Server]
ServerIndex=0
ServerName=服务器名
DBAddr=127.0.0.1        # DBServer地址（本项目移除）
DBPort=6000              # DBServer端口（本项目移除）
IDSAddr=127.0.0.1        # LoginSrv地址（本项目移除）
IDSPort=5600             # LoginSrv端口（本项目移除）
GateAddr=0.0.0.0         # 网关地址（本项目移除）
GatePort=5000            # 网关端口（本项目改为统一端口）
HumLimit=30              # 人类处理限制
MonLimit=30              # 怪物处理限制
UserFull=10000           # 最大在线人数

[Setup]
HomeMap=0                # 出生地图
HomeX=289                # 出生X坐标
HomeY=618                # 出生Y坐标
GroupMembersMax=10       # 组队最大人数
BuildGuild=1000000       # 创建行会费用
GuildWarFee=30000        # 行会战费用
# ... 数百个游戏设置项
```

### 7. `mapinfo.txt` — 地图信息定义

- **格式**: 自定义格式
- **大小**: 88,605 字节，3,831 行

**格式说明**：
```

[0 比奇省 0]
[0122 宫殿 0] DAY
[0137 竞技之地 0] NODRUG

;传送点格式: 源地图 源坐标 -> 目标地图 目标坐标

### 20. `market_def/` — 商人脚本目录

- **文件数**: 100+ 个 .txt 文件

**脚本格式**：
```

# 附录 B：Server Config Converter 工具

## 目标

创建一个 CLI 工具 `cmd/serverconfig/main.go`，将 Delphi 服务端配置文件（`asset/server/`）转换为本项目统一的 JSONC 格式，输出到 `serverconfig/` 目录。

## 背景

- Delphi 原项目有 8 个独立进程（LoginGate、SelGate、RunGate、LoginSrv、DBServer、M2Server 等），各自有独立配置
- 本项目合并为单一 `cmd/server` 进程，需要将分散的配置统一为一套 JSONC 配置
- `asset/server/Map/*.map` 二进制地图文件直接复制，不做转换
- 其余配置文件全部转为 JSONC 格式

## 源文件格式分类

| 格式 | 文件 | 解析策略 |
|------|------|----------|
| **INI** | `!setup.txt`, `Command.ini`, `Exps.ini`, `String.ini`, `GlobalVal.ini`, `系统插件.ini`, `SendSMS.ini` | 通用 INI 解析器 |
| **SQLite** | `数据库/GEEM2.db`（StdItems、Monster、Magic 表） | `database/sql` 读取 |
| **自定义文本** | `mapinfo.txt`, `merchant.txt`, `mongen.txt`, `Npcs.txt`, `GuardList.txt`, `StartPoint.txt`, `MiniMap.txt`, `MakeItem.txt`, `UnbindList.txt`, `FilterItemList.txt`, `GroupItemList.txt`, `ItemRuleList.txt`, `AdminList.txt` 等 | 每个文件专用解析器 |
| **NPC 脚本** | `market_def/*.txt`, `Npc_def/*.txt`, `MapQuest_def/*.txt` | 保持原始脚本格式，包装为 JSON |
| **INI 子目录** | `SmartMonster/*.ini`, `CustomMagic/*.ini`, `MonUseItems/*.txt`, `CustomHeroMagic.ini` | 通用 INI 解析器 |
| **掉落表** | `MonItems/*.txt` | 专用解析器 |
| **城堡** | `Castle/0/SabukW.txt` | INI 解析器 |
| **公告** | `Notice/Notice.txt`, `Notice/LineNotice.txt` | 纯文本包装 |
| **二进制** | `Map/*.map` | 直接复制 |

## 单进程架构设计

### 原 Delphi 8 进程拓扑 → 本项目单进程

| 原进程 | 端口 | 原职责 | 本项目合并方式 |
|--------|------|--------|----------------|
| LoginGate | 7000 | TCP代理，转发到LoginSrv | 合并到 server 监听端口 |
| SelGate | 7100 | TCP代理，转发到DBServer | 合并到 server 监听端口 |
| RunGate | 7200-7900 | TCP代理，转发到M2Server（最多8实例） | 合并到 server 监听端口 |
| LoginSrv | 5500 | 账号认证 | server 内置认证模块 |
| DBServer | 5100/6000 | 角色CRUD | server 内置数据模块（SQLite） |
| M2Server | 5000 | 游戏逻辑 | server 核心（已有） |
| LogDataServer | UDP | 日志收集 | server 内置日志（使用 internal/log） |
| GameCenter | - | 管理控制台 | 不需要（CLI + 日志） |

### 单端口统一监听

server 监听一个端口（默认 7000），内部根据消息内容自动路由：
- 客户端连接后发送 `CM_IDPASSWORD` → 认证流程（原 LoginSrv 功能）
- 认证成功后发送 `CM_QUERYCHR` / `CM_SELCHR` → 角色管理（原 DBServer 功能）
- 选择角色后发送游戏消息 → 游戏逻辑（原 M2Server 功能）

客户端不需要切换连接，全流程在一个 TCP 连接上完成。

### SQLite 数据存储

取代原 Delphi 的平面文件存储（Mir.DB、MirChr.DB、ID.DB）：
- `serverdata/accounts.db` — 账号表（原 LoginSrv 的 ID.DB）
- `serverdata/characters.db` — 角色存档（原 DBServer 的 MirChr.DB）
- `serverdata/guilds.db` — 行会数据（原 GuildBase/）

server.jsonc 中新增数据库配置：
```jsonc
{
  "database": {
    "accounts": "serverdata/accounts.db",
    "characters": "serverdata/characters.db",
    "guilds": "serverdata/guilds.db"
  }
}
```

## 关键设计决策

1. **单端口统一监听** — server 监听一个端口（默认7000），内部自动路由登录/角色/游戏消息，客户端无需切换连接

2. **SQLite 数据存储** — 账号、角色、行会数据统一存储在 SQLite 数据库中，单文件便于备份和管理

3. **GBK 编码处理** — 原始文件为 GBK/GB2312 编码，转换时自动转为 UTF-8

4. **NPC 脚本保持原格式** — 不解析脚本语义，只做包装。后续由服务端脚本引擎直接读取原始格式

5. **空文件跳过** — 大量空文件（Deny*.txt、Disable*.txt 等）不生成对应的 JSONC，只记录到日志

6. **Map 文件直接复制** — 二进制 .map 文件不做转换，直接复制到 `serverconfig/maps/`

7. **SQLite 数据优先** — 物品、怪物、魔法定义以 SQLite 数据库为准（比配置文件更完整）

8. **配置简化** — 原 `!setup.txt` 中的多进程地址/端口配置（DBAddr、IDSAddr、GateAddr 等）全部移除，只保留游戏逻辑配置
