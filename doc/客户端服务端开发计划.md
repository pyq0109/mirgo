## Phase 1: 基础层 — 共享数据 & 协议

**目标**: 将 Delphi `Common/` 目录的所有类型和6Bit协议移植为可复用的Go包

**复杂度**: M | **预计**: 1-2周

### 任务

#### 1.1 `internal/protocol/edcode.go` — 6Bit 编解码
移植 `Common/EDcode.pas`：
- `Encode6BitBuf(src []byte) []byte` — 3字节→4字符，偏移0x3C
- `Decode6BitBuf(src []byte) []byte` — 反向
- `EncodeMessage(msg DefaultMessage) string` / `DecodeMessage(s string) DefaultMessage`
- `EncodeString(s string) string` / `DecodeString(s string) string`
- `EncodeBuffer(buf []byte) string` / `DecodeBuffer(s string, buf []byte)`
- 可选 XOR 掩码表 (`EncodeBitMasks[256]`, `DecodeBitMasks[256]`)

#### 1.2 `internal/protocol/message.go` — 消息类型
移植 `Common/Grobal2.pas` 消息定义：
- `DefaultMessage` 结构体: `Recog int32, Ident uint16, Param uint16, Tag uint16, Series uint16`（16字节，packed）
- `MakeDefaultMsg(ident uint16, recog int32, param, tag, series uint16) DefaultMessage`
- 所有 `CM_*` 常量（客户端→服务端，约50个ID）
- 所有 `SM_*` 常量（服务端→客户端，约100个ID）
- 消息帧格式: `'#' + code(1位数字) + 编码数据 + '!'`

#### 1.3 `internal/protocol/types.go` — 共享数据结构
移植 `Common/Grobal2.pas` 类型：
- 方向常量: `DR_UP(0)` ~ `DR_UPLEFT(7)`
- 网格常量: `UNITX=48, UNITY=32, LOGICALMAPUNIT=40`
- 装备槽常量: `U_DRESS(0)` ~ `U_BUJUK(9)`（10个槽位）
- `StdItem` 结构体（60字节）: Name[20], StdMode, Shape, Weight, AniCount, Looks, DuraMax, AC/MAC/DC/MC/SC, Need/NeedLevel, Price
- `UserItem` 结构体（24字节）: MakeIndex, wIndex, Dura, DuraMax, btValue[14]
- `ClientItem` 结构体: StdItem + MakeIndex + Dura + DuraMax
- `Ability` 结构体（50字节）: Level, AC, MAC, DC, MC, SC, HP, MP, MaxHP, MaxMP, Exp, MaxExp, Weight, MaxWeight 等
- `NakedAbility` 结构体: DC, MC, SC, AC, MAC, HP, MP, Hit, Speed
- `Magic` 结构体: MagicID, Name[12], EffectType, Effect, Spell, Power, TrainLevel[4], MaxTrain[4]
- `UserMagic` 结构体: MagicInfo, Level, MagIdx, TranPoint, Key
- `HumData` / `HumDataInfo` — 完整角色存档结构
- `MsgHeader` 结构体: dwCode(0xAA55AA55), nSocket, wGSocketIdx, wIdent, wUserListIndex, nLength
- `ChrMsg` 结构体: Ident, X, Y, Dir, State, Feature, Saying, Sound
- 毒类型、攻击模式、物品类型等常量

#### 1.4 `internal/protocol/types_test.go` — 编码测试
- 往返测试: `EncodeMessage` → `DecodeMessage`
- 往返测试: `EncodeBuffer` → `DecodeBuffer`（各种大小）
- 验证 `DefaultMessage` 结构体大小 == 16字节
- 验证 `StdItem` 结构体大小 == 60字节
- 验证 `UserItem` 结构体大小 == 24字节

### 交付物
`protocol` 包能编解码所有 MIR2 网络消息。通过基于已知 Delphi 编码字节序列的单元测试验证。

### 关键设计决策
- 使用 `encoding/binary` + `binary.LittleEndian` 进行结构体序列化（Delphi是小端序）
- `StdItem.Name` 使用 `[20]byte`（定长、null终止）保证线路兼容性
- 所有常量定义为类型化枚举

## Phase 2: 客户端 — 窗口 & 场景框架

**目标**: 创建客户端应用壳，包含GLFW窗口、OpenGL上下文、ImGui集成、场景状态机和WIL资源管理器

**复杂度**: M | **预计**: 1-2周

### 任务

#### 2.1 `internal/engine/window.go` — GLFW 窗口
- `Window` 结构体：封装GLFW窗口创建（OpenGL 3.3 Core Profile）
- 可配置大小（默认1024×768）、标题、全屏切换
- 输入回调注册（键盘、鼠标、滚轮）
- 主循环抽象: `Run(updateFn, renderFn)`
- 从 `cmd/mapviewer/main.go` 提取GLFW设置代码

#### 2.2 `internal/engine/gl.go` — OpenGL 状态
- `GLState` 结构体: 单元四边形VAO/VBO、白色纹理、着色器程序
- `UploadTexture(rgba *image.RGBA) uint32`
- `DeleteTexture(id uint32)`
- `DrawQuad(texID uint32, x, y, w, h float32, model, proj mgl32.Mat4)`
- `DrawQuadColor(...)` — 纯色绘制
- `OrthoProj(width, height float32) mgl32.Mat4`
- 从 mapviewer/wilviewer 提取并统一

#### 2.3 `internal/engine/shader.go` — 着色器程序
- `MainShader` — 纹理四边形着色器（pos+uv, model+proj, alpha discard）
- `ColorShader` — 纯色着色器（覆盖层）

#### 2.4 `internal/engine/resourcemanager.go` — WIL 资源缓存
- `ResourceManager` 结构体：持有所有已加载WIL文件
- 字段: `Tiles, SmTiles *wil.File`, `Objects [15]*wil.File`, `Hum, Hair, Weapon *wil.File`, `Mon [28]*wil.File`, `Npc *wil.File`, `Magic, Magic2 *wil.File`, `Items, StateItem, DnItems *wil.File`, `Prguse, Prguse2, Prguse3 *wil.File`, `ChrSel *wil.File`, `Mmap *wil.File`, `Effect, Dragon, Event, HumEffect, MagIcon *wil.File`
- `Load(dataDir string) error` — 从 `asset/client/Data/` 加载所有WIL文件
- 怪物文件延迟加载（Mon1~Mon28）
- 每个WIL文件的纹理缓存: `GetTexture(wilFile *wil.File, index int) uint32`
- LRU淘汰策略

#### 2.5 `internal/engine/scene.go` — 场景状态机
- `SceneType` 枚举: `SceneIntro, SceneLogin, SceneSelectChr, SceneLoginNotice, ScenePlayGame`
- `Scene` 接口: `Open()`, `Close()`, `Update(dt float64)`, `Render()`, `OnKey(key, action)`, `OnMouse(x, y, button, action)`, `OnScroll(x, y float64)`
- `SceneManager` 结构体: `CurrentScene Scene`, `ChangeScene(t SceneType)`
- 每个场景类型的桩实现（渲染占位背景）

#### 2.6 `cmd/client/main.go` — 客户端入口
- 解析CLI参数（数据目录、服务器地址）
- 创建窗口、GL状态、资源管理器
- 初始化场景管理器（`SceneIntro`）
- 主循环: `sceneManager.Update(dt)`, `sceneManager.Render()`, ImGui渲染
- 退出时清理

### 交付物
客户端窗口打开后显示ImGui调试覆盖层，可通过键盘快捷键切换占位场景。WIL资源加载成功（通过日志验证图像数量）。

## Phase 3: 客户端 — 游戏场景 & 地图渲染

**目标**: 使用三层瓦片系统渲染游戏世界，支持相机和坐标变换

**复杂度**: L | **预计**: 2-3周

### 任务

#### 3.1 `internal/engine/camera.go` — 2D 相机
- `Camera` 结构体: X, Y float64, Zoom float32, ViewportW/H int
- `ScreenToWorld(sx, sy) (wx, wy)` — 屏幕→世界坐标
- `WorldToTile(wx, wy) (tx, ty)` — 世界→瓦片坐标
- `TileToScreen(tx, ty) (sx, sy)` — 瓦片→屏幕坐标
- `ViewportTiles() (minX, minY, maxX, maxY)` — 可见瓦片范围
- `CenterOn(tx, ty)` — 居中相机
- `Pan(dx, dy)`, `ZoomAt(factor, cx, cy)`
- `ClampToBounds(mapW, mapH)` — 边界约束
- 从 `cmd/mapviewer/renderer/camera.go` 提取

#### 3.2 `cmd/client/scene_play.go` — 游戏场景（地图渲染）
移植 `PlayScn.pas` 和 `cmd/mapviewer/renderer/renderer.go`：
- `PlayScene` 结构体: mapData, camera, GL状态, resourceManager, actor列表, effect列表
- `DrawTileMap()`:
  - 背景层 (Tiles.wil): stride-2, 2×2块, 48×32瓦片
  - 中间层 (SmTiles.wil): 逐格, 透明
  - 前景层 (Objects.wil): 逐格, 动画帧循环, 门偏移, 混合模式（火焰/灯光用加法混合）
  - 区域系统: 根据格子的 `Area` 字段延迟加载 `Objects{N+1}.wil`
- 坐标变换: `ScreenXYfromMCXY(cx, cy) (sx, sy)`, `CXYfromMouseXY(mx, my) (cx, cy)`
- 视口裁剪: 只渲染屏幕可见瓦片（带边距）

#### 3.3 `cmd/client/scene_play.go` — 移动验证
- `CanWalk(tx, ty int) bool` — 碰撞检测
- `CanRun(fromX, fromY, toX, toY int) bool` — 路径检查
- `CanFly(fromX, fromY, toX, toY int) bool` — 远程攻击路径检查
- `GetDoor(tx, ty) *Door`, `IsDoorOpen(tx, ty) bool`, `OpenDoor(tx, ty)`

#### 3.4 `cmd/client/minimap.go` — 小地图
- 从地图数据生成碰撞纹理（绿色=可行走，灰色=不可通行）
- 渲染小地图覆盖层，带视口矩形
- 点击导航支持
- 从 `cmd/mapviewer/renderer/minimap.go` 提取

### 交付物
客户端渲染完整MIR2地图（三层瓦片），支持相机平移/缩放（WASD + 鼠标滚轮），显示小地图。渲染质量与现有mapviewer一致。

## Phase 4: 客户端 — 角色系统

**目标**: 实现可视化角色系统（玩家、怪物、NPC）及动画

**复杂度**: XL | **预计**: 3-4周

### 任务

#### 4.1 `cmd/client/actor.go` — 角色基类 & 动画模板
移植 `Actor.pas`：
- `ActionInfo` 结构体: Start, Frame, Skip, FTime, UseTick
- `HumanAction` — 14个动作（站立、行走、跑步、左右冲、战斗模式、攻击、重击、大击、烈火准备、施法、坐下、受击、死亡）
- `MonsterAction` — 7个动作（站立、行走、攻击、暴击、受击、死亡、尸体）
- `HumanActions` 模板 — 与Delphi `HA` 完全一致的帧偏移
- `MonsterActions` 模板 — `MA9` ~ `MA47`（39个模板）
- 帧计算: `CalcFrame(action ActionInfo, direction int, frameOffset int) int` → `start + direction*(frame+skip) + offset`
- `WORDER` 数组 — 武器覆盖层排序（男/女各600项）

#### 4.2 `cmd/client/actor.go` — Actor 基础结构体
- 身份: RecogID int32, UserName string, DescUserName string
- 位置: CurrX, CurrY int, Dir int, ShiftX, ShiftY float64（像素偏移，用于平滑移动）
- 外观: Sex, Race, Hair, Dress, Weapon, Job, Appearance uint16
- 状态: Death, Skeleton, WarMode, Grouped bool
- 属性: Abil Ability, Gold int32, HitSpeed int
- 渲染: BodySurface uint32（GL纹理）, BodyOffset int, HairOffset, WeaponOffset int
- 动画: StartFrame, EndFrame, CurrentFrame int, FrameTime, StartTime int64
- 魔法: CurMagic UseMagicInfo, UseMagic bool
- 消息队列: MsgList []ChrMsg（带互斥锁的线程安全）
- 动作: *MonsterAction（指向对应的MA模板）

#### 4.3 `cmd/client/actor_hum.go` — THumActor
移植 `Actor.pas` 中的 THumActor：
- 附加表面: HairSurface, WeaponSurface, HumWinSurface（翅膀）uint32
- `LoadSurface()` — 从 Hum.wil 加载身体, Hair.wil 加载发型, Weapon.wil 加载武器, HumEffect.wil 加载翅膀
- `DrawChr()` — 组合渲染: 身体 + 发型 + 武器 + 翅膀，按 WORDER 覆盖层顺序
- `FeatureChanged()` — 装备变更时重新加载表面
- 帧偏移: `HUMANFRAME = 600`（每性别每装备集）

#### 4.4 `cmd/client/actor_npc.go` — TNpcActor
- `LoadSurface()` — 从 Npc.wil 加载（每NPC 60帧）
- `DrawChr()` — 单表面渲染
- NPC特效表面

#### 4.5 `cmd/client/actor_monster.go` — TMonsterActor
- `LoadSurface()` — 根据外观从 Mon{N}.wil 加载
- `DrawChr()` — 使用正确MA模板的单表面渲染
- 帧数: `MONFRAME=280`, `EXPMONFRAME=360`, `SCULMONFRAME=440`
- Dragon 和 Effect 处理

#### 4.6 `cmd/client/actor_manager.go` — 角色生命周期
- `ActorManager` 结构体: actors map[int32]*Actor（RecogID → Actor）
- `FindActor(id int32) *Actor`
- `FindActorXY(x, y int) *Actor`
- `NewActor(msg DefaultMessage, ...) *Actor` — 从服务器数据创建角色
- `DeleteActor(id int32)`
- `ClearActors()`
- Y轴排序渲染: 按Y坐标排序以正确处理重叠

#### 4.7 `cmd/client/animation.go` — 动画播放器
- `AnimationState` 结构体: currentFrame, startFrame, endFrame, frameTime, lastTick
- `Update(now int64) bool` — 足够时间流逝时推进帧，帧变更返回true
- `SetAction(action ActionInfo, direction int)` — 配置动画
- `Reset()` — 重置到第一帧

### 交付物
角色在地图上正确渲染和动画。可通过调试命令在任意位置创建测试角色（人类、怪物、NPC）。装备覆盖层正确渲染。

## Phase 5: 客户端 — 输入 & 移动

**目标**: 处理玩家输入，实现移动、战斗和UI交互

**复杂度**: M | **预计**: 1-2周

### 任务

#### 5.1 `cmd/client/input.go` — 输入处理器
- `InputHandler` 结构体: 按键状态、鼠标状态、动作队列
- 鼠标处理:
  - 左键: 走到瓦片 / 攻击怪物 / 与NPC交互 / 拾取物品
  - 右键: 跑到瓦片
  - Shift+左键: 强制攻击
  - 中键拖拽: 平移相机
- 键盘处理:
  - F1-F8: 魔法快捷键
  - 1-6: 腰带物品使用
  - B/C/E/G/S/W: 打开面板（背包/角色/装备/行会/状态/魔法）
  - ESC: 关闭面板 / 取消
  - Enter: 聚焦聊天输入
  - 方向键: 相机平移（聊天未聚焦时）

#### 5.2 `cmd/client/action_queue.go` — 动作队列
- `ActionQueue` 结构体: 待执行动作、动作锁状态
- `QueueWalk(dir)`, `QueueRun(dir)`, `QueueHit(dir)`, `QueueSpell(magicID, x, y)`
- 动作锁定: 必须等待服务器响应后才能发送下一个动作（匹配Delphi `ActionLockTime`）
- `ProcessQueue()` — 未锁定时发送下一个动作
- `UnlockAction()` — 服务器确认动作时调用
- 超时处理: 3秒后自动解锁（防卡死）

#### 5.3 `cmd/client/targeting.go` — 目标选择
- `GetCharacter(tx, ty int) *Actor` — 查找瓦片上的角色
- `GetAttackFocusCharacter() *Actor` — 范围内最近敌方
- `GetDropItems(tx, ty int) []GroundItem` — 地面物品
- 目标跟踪: `TargetCret`, `FocusCret`, `MagicTarget` 全局变量

### 交付物
玩家可点击地图行走/跑步，点击测试角色进行目标选择，使用键盘快捷键。移动经过碰撞地图验证。动作锁定防止命令刷屏。

## Phase 6: 客户端 — 网络层

**目标**: 实现客户端TCP网络通信，使用MIR2协议

**复杂度**: L | **预计**: 2周

### 任务

#### 6.1 `internal/netclient/client.go` — TCP 客户端
- `Client` 结构体: conn net.Conn, 接收缓冲区, 发送缓冲区, 互斥锁
- `Connect(addr string) error` — 建立TCP连接
- `Disconnect()`
- `Send(msg DefaultMessage, extra string)` — 编码并发送: `'#' + code + encoded_msg + '!'`
- `SendRaw(data []byte)` — 发送预编码数据
- 读循环（goroutine）: 缓冲传入数据，提取消息帧，分发到处理器
- `OnMessage handler func(msg DefaultMessage, body string)` — 解码消息回调
- 指数退避重连逻辑

#### 6.2 `cmd/client/net_handler.go` — 客户端消息处理器
移植 `ClMain.pas` 的 `DecodeMessagePacket`：
- 登录流程处理器:
  - `SM_CERTIFICATION_SUCCESS` → 请求角色列表
  - `SM_QUERYCHR` → 填充角色选择
  - `SM_NEWCHR_SUCCESS/FAIL` → 角色创建结果
  - `SM_STARTPLAY` → 进入游戏
  - `SM_PASSWD_FAIL` → 显示错误
- 游戏处理器:
  - `SM_NEWMAP`, `SM_CHANGEMAP` → 加载地图
  - `SM_TURN`, `SM_WALK`, `SM_RUN` → 角色移动
  - `SM_HIT`, `SM_STRUCK`, `SM_DEATH` → 战斗视觉
  - `SM_HEAR`, `SM_WHISPER`, `SM_SYSMESSAGE` → 聊天
  - `SM_ADDITEM`, `SM_DELITEM`, `SM_BAGITEMS` → 背包
  - `SM_ADDMAGIC`, `SM_MAGIC_LVEXP` → 魔法列表
  - `SM_OBJECTMOVE` → 角色传送
  - `SM_SHOWEFFECT` → 视觉特效

#### 6.3 `cmd/client/net_send.go` — 客户端消息发送器
移植 `ClMain.pas`：
- `SendLogin(account, password string)`
- `SendNewChr(name string, job, sex int)`
- `SendDelChr(name string)`
- `SendSelChr(name string)`
- `SendSay(text string)`
- `SendWalk(x, y, dir int)`, `SendRun(x, y, dir int)`
- `SendHit(x, y, dir, hitMode int)`, `SendSpell(x, y, dir, magicID int)`
- `SendPickup(x, y int)`, `SendDropItem(itemIdx, count int)`
- `SendTakeOnItem(itemIdx, slot int)`, `SendTakeOffItem(slot int)`
- `SendEat(itemIdx int)`
- `SendMerchantDlgSelect(npcID int, option string)`
- `SendBuyItem(itemIdx, count int)`, `SendSellItem(itemIdx int)`
- `SendRepairItem(itemIdx int)`
- `SendDealTry(targetID int)`, `SendAddDealItem(itemIdx int)`, `SendDealEnd()`
- `SendCreateGroup(memberName string)`, `SendGuildDlg()`

### 交付物
客户端可连接测试服务器，发送登录凭据，接收并显示响应。网络层通过基于已知Delphi编码消息的往返编码测试。

## Phase 7: 服务端 — 核心基础设施

**目标**: 构建M2Server核心：网络引擎、前端引擎、用户引擎和主循环

**复杂度**: XL | **预计**: 3-4周

### 任务

#### 7.1 `internal/netserver/server.go` — TCP 服务器
- `TCPServer` 结构体: 监听器, 接受循环, 连接管理
- `Listen(addr string) error`

## Phase 14: 集成 — 登录→游戏流程

**目标**: 将客户端和服务端连接起来，实现完整的登录到游戏流程（单端口统一架构）

### 任务

#### 14.1 `cmd/server/auth.go` — 认证模块（原 LoginSrv 功能）
- 处理 `CM_IDPASSWORD` 消息，验证账号/密码
- 查询 `serverdata/accounts.db` 数据库
- 返回 `SM_PASSOK_SELECTSERVER` 或 `SM_PASSWD_FAIL`
- 会话管理：SessionID 生成、验证、超时

#### 14.2 `cmd/server/charmgr.go` — 角色管理模块（原 DBServer 功能）
- 处理 `CM_QUERYCHR` 消息，查询角色列表

- 处理 `CM_DELCHR` 消息，删除角色
- 处理 `CM_SELCHR` 消息，选择角色进入游戏
- 查询 `serverdata/characters.db` 数据库
- 返回 `SM_QUERYCHR`, `SM_STARTPLAY` 等响应

#### 14.3 `cmd/server/session.go` — 会话状态机
- 客户端连接状态：`Connected → Authenticated → CharacterSelected → InGame`
- 根据状态路由消息到不同处理器
- 断线重连处理
- 会话超时清理

#### 14.4 `cmd/server/gamedata.go` — 游戏数据初始化
- 从 `serverconfig/` 加载所有 JSONC 配置
- 初始化物品定义、怪物定义、魔法定义
- 加载地图数据
- 刷新初始怪物
- 放置 NPC 和商人

#### 14.5 完整集成测试
- 启动 server 进程（端口 7000）
- 客户端连接到 server:7000
- 登录流程: 认证 → 角色列表 → 选择角色 → 进入游戏
- 玩家出现在地图上，可四处走动
- 其他玩家可见（多客户端时）

### 交付物
端到端登录流程正常工作。玩家可创建账号、创建角色、选择角色并进入游戏世界。移动在客户端间同步。全流程在单一端口上完成。

## 关键设计决策

1. **`internal/protocol/` 集中共享类型** — 客户端和服务端从同一包导入，保证线路兼容性。Delphi在 `Common/Grobal2.pas` 中重复类型；我们集中管理。

2. **`internal/engine/` 共享渲染** — 从mapviewer/wilviewer提取OpenGL样板代码到共享包。客户端和两个查看器都从这里导入。

3. **单端口统一监听** — server 监听一个端口（默认7000），内部自动路由登录/角色/游戏消息，客户端无需切换连接。取代原 Delphi 的 8 进程拓扑。

4. **每连接一个goroutine** — 每个客户端连接有自己的goroutine。通过channel向游戏引擎线程传递消息。

5. **单线程游戏循环** — M2Server主循环以10 ticks/秒在单个goroutine中运行（匹配Delphi的定时器方式）。避免游戏状态的并发修改。重I/O（DB）通过FrontEngine在独立goroutine中运行。

6. **SQLite 数据存储** — 账号、角色、行会数据统一存储在 SQLite 数据库中（`serverdata/` 目录），单文件便于备份和管理。取代原 Delphi 的平面文件存储。

7. **场景接口** — 清晰的UI状态分离。每个场景管理自己的渲染和输入。场景转换是显式的。

8. **客户端动作锁定** — 必须等待服务器确认后才能发送下一个动作。这对防止不同步至关重要，匹配Delphi的反作弊机制。

9. **6Bit编码保真** — 精确重新实现Delphi编码，不做重新设计。确保与现有工具和测试向量的兼容性。

10. **不使用外部游戏引擎** — 自定义OpenGL渲染（匹配现有mapviewer方式）。保持依赖最小化并完全控制渲染管线。
