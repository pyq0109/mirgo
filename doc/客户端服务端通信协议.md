## 1. 总体架构

### 1.1 进程拓扑

```
客户端 ──TCP──► LoginGate(:7000) ──TCP──► LoginSrv(:5500)    [认证]
客户端 ──TCP──► SelGate(:7100)  ──TCP──► DBServer(:5100)     [角色选择]
客户端 ──TCP──► RunGate(:7200+) ──TCP──► M2Server(:5000)     [游戏进行]
```

### 1.2 客户端 Socket 模型

客户端使用**单个 `TClientSocket`**，在不同阶段**重连**到不同服务器：

| 阶段 | `g_ConnectionStep` | 连接目标 |
|---|---|---|
| 登录认证 | `cnsLogin` | LoginGate:7000 |
| 角色选择 | `cnsSelChr` | SelGate:7100 |
| 进入游戏 | `cnsPlay` | RunGate:7200+ |

### 1.3 各层协议总览

| 链路 | 帧格式 | 编码 | 方向 |
|---|---|---|---|
| 客户端 ↔ RunGate | `#...!` 文本 | 6Bit 编码的 TDefaultMessage + 消息体 | 双向 |
| RunGate ↔ M2Server | 二进制 TMsgHeader (magic `$AA55AA55`) | 原始 TDefaultMessage + 编码消息体 | 双向 |
| 客户端 ↔ LoginGate | `#...!` 文本 | 6Bit 编码 | 双向 |
| LoginGate ↔ LoginSrv | `%...$` 文本 | 纯 ASCII | 双向 |
| 客户端 ↔ SelGate | `#...!` 文本 | 6Bit 编码 | 双向 |
| SelGate ↔ DBServer | `%...$` 文本 | 纯 ASCII | 双向 |
| M2Server ↔ DBServer | `#...!` 文本 | 6Bit 编码 + XOR 校验 | 双向 |
| M2Server ↔ LoginSrv | `(...)` 文本 | 纯 ASCII，`/` 分隔字段 | 双向 |

## 2. 6Bit 编解码算法

### 2.1 算法概述

6Bit 编码将 8 位字节数据压缩为 6 位可打印字符，每 3 个输入字节产生 4 个输出字符。

**偏移常量**: `$3C` (十进制 60，字符 `<`)

**输出字符范围**: `$3C` (`<`) 到 `$7B` (`{`)，共 64 个可打印 ASCII 字符。

**压缩比**: 3 字节 → 4 字符 (精确循环)

### 2.2 编码算法详解

源码位置: `Common/EDcode.pas` 第 94-138 行

```pascal
procedure Encode6BitBuf(pSrc, pDest: PChar; nSrcLen, nDestLen: integer);
var
  I, nRestCount, nDestPos: Integer;
  btMade, btCh, btRest: Byte;
begin
  nRestCount := 0;
  btRest := 0;
  nDestPos := 0;
  for I := 0 to nSrcLen - 1 do begin
    if nDestPos >= nDestLen then break;
    btCh := Byte(pSrc[I]);
    btMade := Byte((btRest or (btCh shr (2 + nRestCount))) and $3F);
    btRest := Byte(((btCh shl (8 - (2 + nRestCount))) shr 2) and $3F);
    Inc(nRestCount, 2);
    if nRestCount < 6 then begin
      pDest[nDestPos] := Char(btMade + $3C);
      Inc(nDestPos);
    end else begin
      pDest[nDestPos]     := Char(btMade + $3C);
      pDest[nDestPos + 1] := Char(btRest + $3C);
      Inc(nDestPos, 2);
      nRestCount := 0;
      btRest := 0;
    end;
  end;
  if nRestCount > 0 then begin
    pDest[nDestPos] := Char(btRest + $3C);
    Inc(nDestPos);
  end;
  pDest[nDestPos] := #0;  // null 终止符
end;
```

**逐字节处理过程**:

| 迭代 | nRestCount (前) | 从 btCh 取的位数 | 输出字符数 | 剩余位数 |
|---|---|---|---|---|
| 0 | 0 | 高 6 位 (shr 2) | 1 (btMade) | 2 低位 |
| 1 | 2 | 高 4 位 (shr 4) | 1 (btMade) | 4 低位 |
| 2 | 4 | 高 2 位 (shr 6) | 2 (btMade + btRest) | 0 (重置) |
| 3 | 0 | 高 6 位 (shr 2) | 1 (btMade) | 2 低位 |
| ... | ... | ... | ... | ... |

**3 字节编码示例**:

对于字节 `B0, B1, B2`:

```
迭代 0 (nRestCount=0):
  btMade = (0 OR (B0 >> 2)) AND 0x3F = B0 的高 6 位
  btRest = ((B0 << 6) >> 2) AND 0x3F = B0 的低 2 位，左移 4
  → 输出字符 1，nRestCount=2

迭代 1 (nRestCount=2):
  btMade = (btRest OR (B1 >> 4)) AND 0x3F = (B0低2位<<4) | (B1高4位)
  btRest = ((B1 << 4) >> 2) AND 0x3F = B1 的低 4 位，左移 2
  → 输出字符 2，nRestCount=4

迭代 2 (nRestCount=4):
  btMade = (btRest OR (B1 >> 6)) AND 0x3F = (B1低4位<<2) | (B2高2位)
  btRest = ((B2 << 2) >> 2) AND 0x3F = B2 的低 6 位
  → 输出字符 3 + 字符 4，nRestCount=0 (重置)
```

**结果**: 3 字节 → 4 字符，每个字符 = 6 位数据 + `$3C`。

### 2.3 解码算法详解

源码位置: `Common/EDcode.pas` 第 150-222 行

```pascal
procedure Decode6BitBuf(sSource: PChar; pBuf: PChar; nSrcLen, nBufLen: Integer);
const
  Masks: array[2..6] of byte = ($FC, $F8, $F0, $E0, $C0);
var
  I, nBitPos, nMadeBit, nBufPos: Integer;
  btCh, btTmp, btByte: Byte;
begin
  nBitPos := 2;
  nMadeBit := 0;
  nBufPos := 0;
  btTmp := 0;
  for I := 0 to nSrcLen - 1 do begin
    if Integer(sSource[I]) - $3C >= 0 then
      btCh := Byte(sSource[I]) - $3C    // 去除 $3C 偏移
    else begin
      nBufPos := 0;                      // 无效字符：中止
      break;
    end;
    if nBufPos >= nBufLen then break;
    if (nMadeBit + 6) >= 8 then begin
      // 累积位数足够形成完整字节
      btByte := Byte(btTmp or ((btCh and $3F) shr (6 - nBitPos)));
      pBuf[nBufPos] := Char(btByte);
      Inc(nBufPos);
      nMadeBit := 0;
      if nBitPos < 6 then Inc(nBitPos, 2)
      else begin
        nBitPos := 2;
        continue;    // 跳过 btTmp 计算
      end;
    end;
    btTmp := Byte(Byte(btCh shl nBitPos) and Masks[nBitPos]);
    Inc(nMadeBit, 8 - nBitPos);
  end;
  pBuf[nBufPos] := #0;
end;
```

**解码掩码表**:

| nBitPos | Mask | 二进制 |
|---|---|---|
| 2 | `$FC` | `11111100` |
| 3 | `$F8` | `11111000` |
| 4 | `$F0` | `11110000` |
| 5 | `$E0` | `11100000` |
| 6 | `$C0` | `11000000` |

**nBitPos 循环**: 2 → 4 → 6 → 2 → 4 → 6 → ...

| nBitPos | 行为 |
|---|---|
| 2 | 若位数足够：组合当前字符的 2 位 + 6 位余数。存储余数 `(btCh << 2) AND 0xFC`。nBitPos → 4 |
| 4 | 若位数足够：组合当前字符的 4 位 + 4 位余数。存储余数 `(btCh << 4) AND 0xF0`。nBitPos → 6 |
| 6 | 组合当前字符的 6 位 + 2 位余数。存储余数 `(btCh << 6) AND 0xC0`。nBitPos → 2，`continue` |

### 2.4 高层封装函数

| 函数 | 输入 | 输出 | 说明 |
|---|---|---|---|
| `EncodeMessage(msg)` | TDefaultMessage (12 字节) | 16 字符字符串 | 结构体 → 6Bit 编码 |
| `DecodeMessage(str)` | 16 字符字符串 | TDefaultMessage | 6Bit 解码 → 结构体 |
| `EncodeString(str)` | 原始字符串 | 6Bit 编码字符串 | 字符串 → 6Bit 编码 |
| `DecodeString(str)` | 6Bit 编码字符串 | 原始字符串 | 6Bit 解码 → 字符串 |
| `EncodeBuffer(buf, size)` | 原始字节缓冲区 | 6Bit 编码字符串 | 二进制 → 6Bit 编码 |
| `DecodeBuffer(src, buf, size)` | 6Bit 编码字符串 | 原始字节缓冲区 | 6Bit 解码 → 二进制 |

**编码后长度计算**:
- `GetCodeMsgSize(n)` = `n * 4 / 3` 向上取整
- TDefaultMessage (12 字节) → `ceil(12 * 4 / 3)` = 16 字符 (`DEFBLOCKSIZE`)

### 2.5 OLDMODE vs NEWMODE

代码中有编译时开关 `ENDECODEMODE`:

- **OLDMODE (0)**: 纯 6Bit 编解码，**无混淆**。这是实际使用的模式 (`ENDECODEMODE = OLDMODE`)。
- **NEWMODE (1)**: 在 6Bit 编解码前后添加 XOR 混淆，使用 `EncodeBitMasks[]`/`DecodeBitMasks[]` 查找表和多个 XOR 密钥。

**NEWMODE 密钥**:
```
n4CEEF4 = $408D4D
n4CEEF8 = $0C08BA52E
w4CEF00 = $8D34
```

**标准热血传奇游戏使用 OLDMODE，无 XOR 混淆。**

## 3. 核心数据结构

### 3.1 TDefaultMessage (12 字节)

所有客户端-服务端消息的基本封装。

```
偏移    大小    字段      说明
------  ----  --------  ----------------------------------
0x00    4     Recog     认证/识别码 (Integer)
0x04    2     Ident     消息 ID (Word, CM_*/SM_* 常量)
0x06    2     Param     参数 1 (Word)
0x08    2     Tag       参数 2 (Word)
0x0A    2     Series    参数 3 (Word)
```

**总大小: 12 字节**。6Bit 编码后为 16 字符 (`DEFBLOCKSIZE = 16`)。

### 3.2 TMsgHeader (16 字节)

RunGate ↔ M2Server 二进制协议的帧头。

```
偏移    大小    字段            说明
------  ----  --------------  ----------------------------------
0x00    4     dwCode          魔数: $AA55AA55 (RUNGATECODE)
0x04    4     nSocket         客户端 socket 句柄 (标识玩家连接)
0x08    2     wGSocketIdx     Gate 侧会话索引 (0-based)
0x0A    2     wIdent          消息类型 (GM_* 常量)
0x0C    2     wUserListIndex  M2Server 侧用户列表索引 (1-based)
0x0E    4     nLength         载荷长度（字节数，可为负数）
```

**总大小: 16 字节**。字节序: 小端 (x86 原生)。

### 3.3 GM_* 消息类型

| 常量 | 值 | 方向 | 说明 |
|---|---|---|---|
| `GM_OPEN` | 1 | Gate→M2 | 新客户端连接，载荷 = 客户端 IP 字符串 |
| `GM_CLOSE` | 2 | Gate→M2 | 客户端断开，无载荷 |
| `GM_CHECKSERVER` | 3 | M2→Gate | 心跳（M2Server→RunGate） |
| `GM_CHECKCLIENT` | 4 | Gate→M2 | 心跳（RunGate→M2Server） |
| `GM_DATA` | 5 | 双向 | 游戏数据消息 |
| `GM_SERVERUSERINDEX` | 6 | M2→Gate | 分配用户列表索引 |
| `GM_RECEIVE_OK` | 7 | 双向 | 流控确认 |
| `GM_TEST` | 20 | M2→Gate | 负载测试 |

### 3.4 TCharDesc (8 字节)

角色外观描述，用于移动/动作消息。

```
偏移    大小    字段      说明
------  ----  --------  ----------------------------------
0x00    4     Feature   外观编码 (Race|Appr|Hair|Dress|Weapon)
0x04    4     Status    状态标志/增益效果
```

### 3.5 TMessageBodyW (8 字节)

Word 扩展消息体。

```
偏移    大小    字段    说明
------  ----  ------  --------
0x00    2     Param1  参数 1
0x02    2     Param2  参数 2
0x04    2     Tag1    标签 1
0x06    2     Tag2    标签 2
```

### 3.6 TMessageBodyWL (16 字节)

Integer 扩展消息体。

```
偏移    大小    字段     说明
------  ----  -------  --------
0x00    4     lParam1  参数 1
0x04    4     lParam2  参数 2
0x08    4     lTag1    标签 1
0x0C    4     lTag2    标签 2
```

### 3.7 TShortMessage (4 字节)

```
偏移    大小    字段    说明
------  ----  ------  --------
0x00    2     Ident   消息 ID
0x02    2     wMsg    消息值
```

## 4. 消息帧格式

### 4.1 客户端 → 服务器 (`#...!` 帧)

源码位置: `Client/ClMain.pas` 第 2805-2816 行

```
# <1位序列号> <载荷> !
```

- `#` — 起始定界符 (0x23)
- `code` — 单位数字 1-9，循环递增（用于排序/去重）
- 载荷 — 编码后的数据
- `!` — 结束定界符 (0x21)

```pascal
procedure TfrmMain.SendSocket(sendstr: string);
const
   code: byte = 1;
begin
   if CSocket.Socket.Connected then begin
      CSocket.Socket.SendText('#' + IntToStr(code) + sendstr + '!');
      Inc(code);
      if code >= 10 then code := 1;
   end;
end;
```

### 4.2 载荷组装模式

**模式 1: 仅 TDefaultMessage** (用于简单消息)
```
EncodeMessage(msg)                          → 16 字符
```

**模式 2: TDefaultMessage + 编码字符串** (用于携带文本参数)
```
EncodeMessage(msg) + EncodeString(text)     → 16 字符 + N 字符
```

**模式 3: TDefaultMessage + 编码缓冲区** (用于携带二进制结构体)
```
EncodeMessage(msg) + EncodeBuffer(buf, len) → 16 字符 + N 字符
```

**模式 4: 仅编码字符串** (用于 RunLogin 等特殊消息)
```
EncodeString(text)                          → N 字符
```

### 4.3 服务器 → 客户端 (`#...!` 帧)

**注意**: 服务器→客户端的消息**没有序列号**。

```
# <6Bit 编码的 TDefaultMessage> [<已编码的消息体>] !
```

RunGate 将 M2Server 的二进制消息转换为文本格式:

```pascal
// nLength < 0: 纯文本消息
sSendMsg := '#' + String(Buffer) + '!';

// nLength >= SizeOf(TDefaultMessage): TDefaultMessage + 消息体
sSendMsg := '#' + EncodeMessage(pDefMsg^) + String(PChar(@Buffer[SizeOf(TDefaultMessage)])) + '!';

// nLength == SizeOf(TDefaultMessage): 仅 TDefaultMessage
sSendMsg := '#' + EncodeMessage(pDefMsg^) + '!';
```

**重要**: 消息体部分从 M2Server 到达时已经是 6Bit 编码的，RunGate **不会重新编码**，只重新编码 TDefaultMessage 结构体。

### 4.4 控制消息前缀

客户端还处理以下非标准消息前缀:

**`+` 前缀** (战斗技能控制):
```
+GOOD   → 动作确认 (ActionLock := FALSE)
+FAIL   → 动作失败
+PWR    → 启用 PowerHit (攻杀)
+LNG    → 启用 LongHit (刺杀)
+ULNG   → 禁用 LongHit
+WID    → 启用 WideHit (半月弯刀)
+UWID   → 禁用 WideHit
+CRS    → 启用 CrsHit (十字斩)
+UCRS   → 禁用 CrsHit
+TWN    → 启用 TwnHit (开天斩)
+UTWN   → 禁用 TwnHit
+FIR    → 启用 FireHit (烈火剑法)
+UFIR   → 禁用 FireHit
```

**`=` 前缀** (特殊信号):
```
=DIG    → 设置挖掘标志
```

这些控制消息**不经过 6Bit 编码**，是纯 ASCII 文本。

### 5.2 各阶段消息详情

#### 阶段 1: 登录认证 (LoginGate)

**客户端发送**:
```pascal
msg := MakeDefaultMsg(CM_IDPASSWORD, 0, 0, 0, 0);
SendSocket(EncodeMessage(msg) + EncodeString(uid + '/' + passwd));
```

CM_IDPASSWORD = 2001，消息体 = `EncodeString("账号/密码")`

**服务器响应**:
- `SM_PASSOK_SELECTSERVER (529)` — 密码正确，显示服务器列表
- `SM_PASSWD_FAIL (503)` — 密码错误
- `SM_ID_NOTFOUND (502)` — 账号不存在

#### 阶段 2: 选择服务器 (LoginGate)

**客户端发送**:
```pascal
msg := MakeDefaultMsg(CM_SELECTSERVER, 0, 0, 0, 0);
SendSocket(EncodeMessage(msg) + EncodeString(svname));
```

**服务器响应**: `SM_SELECTSERVER_OK (530)`，消息体 = `"runaddr/runport/certstr"`

#### 阶段 3: 查询角色 (SelGate)

**客户端发送**:
```pascal
msg := MakeDefaultMsg(CM_QUERYCHR, 0, 0, 0, 0);
SendSocket(EncodeMessage(msg) + EncodeString(LoginId + '/' + IntToStr(Certification)));
```

**服务器响应**: `SM_QUERYCHR (520)`，消息体格式:
```
*name1/job1/hair1/level1/sex1/name2/job2/hair2/level2/sex2/
```
`*` 前缀表示上次选中的角色。

#### 阶段 4: 选择角色 (SelGate)

**客户端发送**:
```pascal
msg := MakeDefaultMsg(CM_SELCHR, 0, 0, 0, 0);
SendSocket(EncodeMessage(msg) + EncodeString(LoginId + '/' + chrname));
```

**服务器响应**: `SM_STARTPLAY (525)`，消息体 = `"runaddr/runport"`

#### 阶段 5: 游戏登录 (RunGate)

**客户端发送** (无 TDefaultMessage，纯编码字符串):
```pascal
sSendMsg := format('**%s/%s/%d/%d/%d', [LoginId, CharName, Certification, CLIENT_VERSION_NUMBER, RUNLOGINCODE]);
SendSocket(EncodeString(sSendMsg));
```

格式: `**<账号>/<角色名>/<认证码>/<客户端版本>/<标志>`

常量: `CLIENT_VERSION_NUMBER = 120040918`, `RUNLOGINCODE = 9`

**服务器响应序列**:
1. `SM_SENDNOTICE (658)` — 游戏公告/协议文本
2. 客户端确认 `CM_LOGINNOTICEOK (1018)`
3. `SM_NEWMAP (51)` — 地图名, 玩家 X/Y
4. `SM_LOGON (50)` — 玩家外观, 状态 → 场景切换到 `stPlayGame`
5. `SM_ABILITY (52)` — 玩家属性
6. `SM_SUBABILITY (752)` — 子属性

## 6. 进程间通信协议

### 6.1 RunGate ↔ M2Server 二进制协议

#### 消息封装格式

```
+------------------+------------------+------------------+
| TMsgHeader (16B) | TDefaultMsg (12B)| 编码消息体 (可选) |
| dwCode=$AA55AA55 | Recog/Ident/...  |                  |
| wIdent=GM_DATA   |                  |                  |
+------------------+------------------+
```

#### nLength 字段语义

| nLength | 含义 |
|---|---|
| `> 0` | 载荷 = TDefaultMessage (12 字节) + 可选编码消息体 |
| `< 0` | 载荷 = 纯文本字符串 (无 TDefaultMessage) |
| `= 0` | 无载荷 (心跳等控制消息) |

#### GM_OPEN 流程

1. RunGate 发送新客户端连接通知:
```
TMsgHeader { dwCode=$AA55AA55, nSocket=<socket句柄>, wIdent=GM_OPEN, nLength=len+1 }
+ null 终止的客户端 IP 字符串
```

2. M2Server 分配用户列表槽位，响应:
```
TMsgHeader { dwCode=$AA55AA55, nSocket=<socket句柄>, wGSocketIdx=<gate槽位>,
             wIdent=GM_SERVERUSERINDEX, wUserListIndex=<1-based索引>, nLength=0 }
```

#### GM_CLOSE 流程

RunGate 发送:
```
TMsgHeader { dwCode=$AA55AA55, nSocket=<socket句柄>, wIdent=GM_CLOSE, nLength=0 }
```

#### GM_DATA 协议转换

**客户端 → M2Server (入站)**:

1. 客户端发送: `#<packet_idx><6Bit编码数据>!`
2. RunGate 解析 `#...!` 帧
3. 首次消息 (认证): 整个 `#...!` 消息原样封装在 GM_DATA 中
4. 后续消息: RunGate 解码 TDefaultMessage，重组为二进制格式:
```
TMsgHeader { wIdent=GM_DATA, nSocket=<socket>, wUserListIndex=<idx> }
+ [TDefaultMessage 原始字节 (12 字节)]
+ [编码消息体字符串 + #0]  (可选)
```

**M2Server → 客户端 (出站)**:

1. M2Server 发送二进制:
```
TMsgHeader { wIdent=GM_DATA, nSocket=<socket>, nLength=<载荷长度> }
+ 载荷
```

2. RunGate 转换为文本:
```pascal
if nLength < 0 then
  '#' + String(Buffer) + '!'
else if nLength > SizeOf(TDefaultMessage) then
  '#' + EncodeMessage(DefMsg) + String(body) + '!'
else
  '#' + EncodeMessage(DefMsg) + '!'
```

#### 消息边界解析

M2Server 在 `RunSock.pas` 的 `ExecGateBuffers` 中解析消息:

1. 累积接收到的字节到 `Gate.Buffer`
2. 当缓冲区 >= 16 字节 (SizeOf(TMsgHeader)) 时:
   - 读取 TMsgHeader
   - 验证: `dwCode == RUNGATECODE` 且 `abs(nLength) < $8000`
   - 若数据不足，等待更多数据
   - 调用 `ExecGateMsg` 处理消息
   - 缓冲区前进 `SizeOf(TMsgHeader) + nLength`
3. 若魔数不匹配，前进 1 字节重试 (重新同步)

### 6.2 LoginGate ↔ LoginSrv 文本协议

#### 帧格式: `%...$`

| 格式 | 说明 |

## 9. 流控与心跳机制

### 9.1 客户端 ↔ Gate 流控 (`*` 字符)

所有三种 Gate (LoginGate, SelGate, RunGate) 实现双向流控:

**Gate → 客户端**:
- 当 `nCheckSendLength >= SENDCHECKSIZE` (250-512 字节) 时，Gate 在下一个发送块前添加 `*`
- 当 `nCheckSendLength >= SENDCHECKSIZEMAX` (2048) 时，Gate 设置 `boSendAvailable := False`，暂停发送 `dwClientCheckTimeOut` (3 秒)

**客户端 → Gate**:
- 客户端收到 `*` 后，在下一条消息中回复 `*`
- Gate 收到 `*` 后，重置 `boSendAvailable`, `boSendCheck`, `nCheckSendLength`

**客户端侧处理** (`ClMain.pas` CSocketRead):
```pascal
n := pos('*', data);
if n > 0 then begin
   data2 := Copy(data, 1, n-1);
   data := data2 + Copy(data, n+1, Length(data));
   CSocket.Socket.SendText('*');    // 心跳响应
end;
```

### 9.2 RunGate ↔ M2Server 心跳

**RunGate → M2Server** (每 2 秒):
```
TMsgHeader { wIdent=GM_CHECKCLIENT, nLength=0 }
```

**M2Server → RunGate**:
```
TMsgHeader { wIdent=GM_CHECKSERVER, nLength=0 }
```

**超时检测**: RunGate 追踪 `dwCheckServerTick`，若 `GetTickCount - dwCheckServerTick > 3分钟`，标记连接失败并关闭。

### 9.3 LoginGate ↔ LoginSrv 心跳

- LoginGate 每 2 秒发送 `%--$`
- LoginSrv 响应 `%++$`
- 超时: 10 秒无响应 → `boKeepAliveTimcOut := True`，关闭连接

### 9.4 M2Server ↔ LoginSrv 心跳

- M2Server 定期发送: `(SS_SERVERINFO/<服务器名>/<索引>/<在线人数>)`
- LoginSrv 响应: `(SS_KEEPALIVE/<全局总在线>)`
- 超时: 30 秒无更新 → 标记服务器离线

### 9.5 速度hack检测

服务器发送 `SM_TIMECHECK_MSG (810)`，`msg.Recog` 包含服务器时间戳。客户端将其与本地 `GetTickCount` 比较以检测速度作弊。

```go
// 核心编解码
```


### 10.2 消息帧解析器

需要实现:
```go
// 解析 #...! 帧
func ParseFrame(data []byte) (frame []byte, remaining []byte)

// 从帧中提取消息
func ParseMessage(frame []byte) (msg TDefaultMessage, body []byte)

// 组装发送帧

```

### 10.3 TMsgHeader 解析

```go
type TMsgHeader struct {
    DwCode         uint32  // 0xAA55AA55
    NSocket        int32
    WGSocketIdx    uint16
    WIdent         uint16
    WUserListIndex uint16
    NLength        int32
}
// 总大小: 16 字节，小端序
```

### 10.4 进程间协议适配

在单进程架构中，不需要 LoginGate/SelGate/RunGate 的代理协议，但需要:
1. 客户端连接时解析 `#...!` 帧格式
2. 直接处理 TDefaultMessage + 消息体
3. 实现 `*` 流控机制
4. 实现心跳机制

### 10.5 消息体解码规则

不同 SM_* 消息使用不同的消息体格式:

| 消息类型 | 消息体解码方式 |
|---|---|
| SM_TURN, SM_WALK, SM_RUN 等移动动作 | `DecodeBuffer(body, TCharDesc)` (8 字节) |
| SM_STRUCK | `DecodeBuffer(body, TMessageBodyWL)` (16 字节) |
| SM_HEAR, SM_SYSMESSAGE 等聊天 | `DecodeString(body)` (文本) |
| SM_LOGON | `DecodeBuffer(body, TMessageBodyWL)` (16 字节) |
| SM_ABILITY | `DecodeBuffer(body, TAbility)` (50 字节) |
| SM_BAGITEMS | 多次 `DecodeBuffer`，以 `/` 分隔 |
| SM_ADDITEM | `DecodeBuffer(body, TClientItem)` (68 字节) |

**复合消息体**: 某些消息的 body = 二进制结构体 + 编码字符串:
```pascal
// 先解码二进制部分
DecodeBuffer(body, @desc, sizeof(TCharDesc));
// 若有剩余，解码字符串部分 (名字/颜色等)
if Length(body) > GetCodeMsgSize(sizeof(TCharDesc)*4/3) then begin
   Body2 := Copy(Body, GetCodeMsgSize(sizeof(TCharDesc)*4/3)+1, Length(body));
   data := DecodeString(body2);
end;
```

| 字段 | 大小 | 通用用途 |

